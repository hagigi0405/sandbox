# サンドボックス開発計画

Java 21, Spring Boot 3, PostgreSQL, Thymeleaf を使用した、実践的なWebアプリケーション開発の学習計画です。

---

## 📋 要件概要
- ログイン機能なしのシンプルな「タスク管理（Todo）アプリケーション」の開発
- APIファースト設計（Swagger/OpenAPI仕様）の学習と実践
- Dockerを使用したデータベース（PostgreSQL）の構築と接続
- Thymeleaf画面からJavaScriptでREST APIを呼び出す非同期通信の実装（アプローチ②）
- 自動ビルドとDockerによるデプロイ（家庭内LAN公開）まで

---

## 🛠 確定技術スタック

| レイヤー | 使用技術 | 備考 |
| :--- | :--- | :--- |
| **言語** | Java 21 | 最新の文法や機能を意識して活用する |
| **フレームワーク** | Spring Boot 3 | Spring MVC / Spring Data JPA 等を使用 |
| **データベース** | PostgreSQL | Dockerコンテナ上で動作させ、ローカル環境を汚さない構成にする |
| **フロントエンド** | Thymeleaf + JS | 画面はThymeleafで構成し、非同期通信（fetch API）でAPIと連携する |
| **API仕様定義** | OpenAPI (Swagger) | 実装前にJSON形式のAPI仕様を策定・Fixさせる |
| **バージョン管理** | Git / GitHub | Conventional Commitsに準拠したコミットログを意識する |
| **開発環境** | VSCode (Antigravity IDE) | ペアプログラミングをフル活用 |

## 🗺 学習ロードマップ（マイルストーン）

### 【ステップ 1】環境構築とGit設定 ✅
- **作業内容**:
  1. Gitの `user.name` と `user.email` を設定する。（完了）
  2. `g:\work\sandbox` にワークスペースを切り替え、リポジトリとGitHubを接続する。（完了）
  3. Spring Boot 3プロジェクトを初期化し、雛形を展開する。（完了）

### 【ステップ 2】DockerによるDB環境構築 🐳（現在ここ！）
- **作業内容**:
  1. `docker-compose.yml` を作成し、PostgreSQLコンテナを立ち上げる。
  2. データベース接続クライアント（VSCode拡張機能やDBeaverなど）から接続確認を行う。
  3. Spring Bootの `application.yml` にデータベース接続設定を記述する。

### 【ステップ 3】APIファースト設計 📄 ✅
- **作業内容**:
  1. タスク管理に必要なAPI（作成・取得・更新・削除）を `openapi.yaml` で定義し、仕様をFixさせる。（完了）

### 【ステップ 4】データベース設計とビジネスロジック実装 💾
- **作業内容**:
  1. タスク情報を格納するテーブルを設計し、PostgreSQLに適用する。
  2. Spring Data JPAを使用してエンティティ、リポジトリ層、サービス層、コントローラー層（REST API）を実装する。
  3. ユニットテストを作成し、API機能の堅牢性を保証する。

### 【ステップ 5】ThymeleafとJSによるフロントエンド実装 🎨
- **作業内容**:
  1. タスク管理画面のHTMLをThymeleafで作成する。
  2. JavaScript（fetch API）を用いて、追加・更新・削除などの操作を非同期でJava APIと連携させる。
  3. 動的なUI描画や簡易バリデーションを実装する。

### 【ステップ 6】ビルドとデプロイ 🚀
- **作業内容**:
  1. Spring Bootアプリケーションのビルド（Gradle）を実行する。
  2. Dockerfileを作成し、アプリをコンテナ化する。
  3. 家庭内LANで公開し、別端末からアクセスできるか検証する。

# Git初期設定と初回コミットの流れ

このメモは、この環境（Windows + PowerShell）でGitの初期設定から初回コミットを行うための手順だよ！

---

## 1. 文字化け防止の設定（重要！）
Windows環境のPowerShellターミナルで日本語が文字化けするのを防ぐために、ターミナルを開いたらまず以下のコマンドを実行してね！
```bash
chcp 65001
```

## 2. Gitのユーザー設定
コミットするためには、誰がコミットしたかをGitに教えてあげる必要があるよ。以下のコマンドで設定してね！

```bash
# ユーザー名を設定する（例: "hikasha"）
git config user.name "あなたのユーザー名"

# メールアドレスを設定する（例: "hikasha@example.com"）
git config user.email "あなたのメールアドレス"
```

### 設定を確認するコマンド
正しく設定できたか確認したいときは、以下のコマンドを実行してみてね！
```bash
git config user.name
git config user.email
```

## 3. ファイルをステージングする
コミットしたいファイルを「ステージングエリア」という準備場所に登録するよ。今回は `README.md` を登録してみよう！
```bash
git add README.md
```

> [!NOTE]
> **💡 SVNの「追加(add)」との違い**
> * **新規ファイル**: SVNと同じく、新しく作ったファイルを管理下に入れるよ！
> * **既存ファイルの変更**: SVNと違って、既存ファイルの変更もコミットする前に毎回 `git add` して「準備台」に載せる必要があるよ！
>
> つまりGitでは、`git add`（準備台に載せる） ➔ `git commit`（記録する）の2ステップが必要になるんだね！

もし新しく作ったファイルなどもまとめて全部ステージングしたいときは、以下のコマンドでも大丈夫だよ！
```bash
git add .
```

### 状態を確認するコマンド
どのファイルがステージングされているか確認するにはこれ！
```bash
git status
```

## 4. 初回コミットを実行する！
ステージングしたファイルをコミット（記録）するよ！
メッセージの書き方は、変更の種類がわかりやすいように `chore:` や `docs:` などのプレフィックスをつけるのがおすすめだよ（Conventional Commits形式）。

```bash
git commit -m "chore: initial commit with README"
```

これで初回コミットは完了だよ！お疲れ様！✨

---

## 💡 ターミナルの出力メッセージを日本語にする方法
Gitコマンドの出力（英語）が読みにくい場合は、Git Bashの設定で日本語表示に変更できるよ！

ターミナル（Git Bash）で以下のコマンドを実行してね：

```bash
# 次回以降もずっと日本語表示にする設定を追加する
echo "export LANG=ja_JP.UTF-8" >> ~/.bash_profile

# 設定を今すぐ反映させる
source ~/.bash_profile
```

設定したあとに `git status` などを打つと、メッセージが日本語で案内されるようになるから分かりやすくなるよ！

## 5. GitHubに接続してプッシュする
ローカルでコミットした内容をGitHubに保存（プッシュ）するための手順だよ！

### ① GitHub上で空のリポジトリを作成する
1. GitHubにログインして、新しいリポジトリ（New repository）を作成するよ！
2. リポジトリ名は `sandbox` などにしてね。
3. **【超重要】**：「README」や「.gitignore」などの初期化ファイル作成チェックボックスは、**全部オフ（チェックしない）**にして、「Create repository」をクリックしてね！すでに手元にあるファイルと衝突しちゃうのを防ぐためだよ。

### ② リモートリポジトリ（GitHub）と紐付ける
リポジトリを作成すると、画面に `https://github.com/〜` で始まるURLが表示されるから、それをコピーして以下のコマンドを実行してね！

```bash
# リモートリポジトリを「origin」という名前で登録する
git remote add origin <コピーしたGitHubのURL>
```
※URLの例：`https://github.com/あなたのユーザー名/sandbox.git`

### ③ デフォルトブランチの名前を「main」にする
すでに `main` になっているけど、念のため以下のコマンドでブランチ名を確定させるよ！
```bash
git branch -M main
```

### ④ GitHubへプッシュ（アップロード）する！
いよいよGitHubにファイルを送るよ！
```bash
# 初回は「-u origin main」をつけてプッシュするよ
git push -u origin main
```
※このコマンドを実行すると、ブラウザが開いてGitHubへのサインイン（認証）を求められるから、画面の指示に従ってログインを承認してね！
※2回目以降は、シンプルに `git push` と打つだけでプッシュできるようになるよ！
